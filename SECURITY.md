# Política de Seguridad

## Versiones Soportadas

Actualmente solo se proporciona soporte de seguridad para la **última versión publicada**. Las versiones anteriores no reciben actualizaciones de seguridad.

| Versión | Soportada |
|---|---|
| Última | ✅ |
| Anteriores | ❌ |

## Cómo Reportar una Vulnerabilidad

Si encuentras una vulnerabilidad de seguridad, **no abras un *issue* público**. En su lugar, utiliza uno de los siguientes canales:

1. **GitHub Security Advisory**: ve a la pestaña *Security* del repositorio y crea un *advisory* privado.
2. **Correo electrónico**: escribe a los mantenedores a través del sistema de *issues* de GitHub solicitando un medio de contacto privado.
3. **Issue privado**: si no hay otra opción, abre un *issue* marcándolo como confidencial (si tu plan de GitHub lo permite).

### Qué Incluir en el Reporte

- Descripción clara del problema y su posible impacto.
- Pasos para reproducir la vulnerabilidad o PoC (*proof of concept*).
- Versión del software afectada.
- Si es posible, una sugerencia de mitigación o solución.

## Respuesta Esperada

- **Confirmación de recepción**: dentro de las 72 horas hábiles posteriores al reporte.
- **Evaluación inicial**: dentro de los 5 días hábiles.
- **Parche o mitigación**: se publicará tan pronto como esté lista, dependiendo de la gravedad y complejidad.
- **Divulgación pública**: coordinaremos la divulgación contigo una vez que el parche esté disponible.

Agradecemos tu ayuda para mantener este proyecto seguro.
